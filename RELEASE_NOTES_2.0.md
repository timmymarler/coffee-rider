# Coffee Rider v2.0 Release Notes

## Major Features

### Groups & Route Sharing
- Create and manage riding groups (Pro feature)
- Invite members via email
- Share saved routes with groups or publicly
- Three visibility levels: Private, Group, and Public
- View all shared routes in the Groups tab
- Filter routes by sharing status in Saved Routes

### Route to Home
- Set home address in Profile
- Long press Follow Me button to create route to home
- Auto-activates Follow Me mode when routing home
- Visual destination marker with home icon
- Integrated with waypoints system

### Enhanced Route Management
- Improved route filtering with persistent controls
- View public routes from all users
- Better empty state messaging
- Share routes with visibility controls

## Improvements

### UI/UX Enhancements
- Fixed Follow Me staying active during programmatic zoom
- Fixed Follow Me disabling on manual map pan
- Fixed filter navigation - filters remain accessible when results are empty
- Added group badges on shared route cards
- Added public route indicators
- Improved modal positioning and styling

### Documentation
- Comprehensive Help screen updates
- Groups and Shared Routes documentation
- Route to Home feature guide
- Updated user capabilities by role

## Bug Fixes
- Fixed filter navigation issue when no shared routes exist
- Fixed public routes not being visible to other users
- Fixed Follow Me being disabled during route-to-home
- Fixed button prop naming issues
- Fixed icon fallbacks for better compatibility

## Technical Updates
- Multi-query route fetching for better performance
- Improved Firestore rules for group route queries
- Better error handling for offline scenarios
- Enhanced geocoding integration
- Optimized real-time listeners
